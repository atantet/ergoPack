7 ergoPack is a collection of scientific routines in C++

15 ============
16 
17 Getting the code {#getting-the-code}
18 ----------------
19 
20 First the ergoPack repository should be cloned using [git].
21 To do so:
22 1. Change directory to where you want to clone the repository $GITDIR:
23 
24  cd $GITDIR
25 
26 2. Clone the ergoPack repository (git should be installed):
27 
28  git clone https://github.com/atantet/ergoPack
29 
30 Dependencies {#dependencies}
31 ------------
32 
33 Mandatory libraries:
34 - [GSL] is used as the main C scientific library.
35 - [Eigen] is a C++ template library for linear algebra, mainly used for sparse matrices manipulation.
36 
37 Specific libraries:
38 - [ARPACK++] is an object-oriented version of the ARPACK package used to calculate the spectrum of sparse matrices in atspectrum.hpp.
39 - [OpenMP][OMP] is used for multi-threading by transferOperator.hpp
40 when WITH_OMP is set to 1 when compiling.
41 
42 Installing the code {#installing-the-code}
43 -------------------
44 
45 1. Create a directory ergoPack/ in your favorite include directory $INCLUDE:
46 
47  mkdir $INCLUDE/ergoPack
48 
49 2. Copy the ergoPack/*.hpp source files to $INCLUDE/ergoPack/:
50 
51  cd $GITDIR/ergoPack
52  cp *.hpp $INCLUDE/ergoPack
53 
54 3. Include these files in your C++ codes. For example, in order to include the matrix manipulation functions in atmatrix.hpp,
55 add in your C++ file:
56 
57  #include <ergoPack/atmatrix.hpp>

73 Compiling {#compiling}
74 =========
75 
76 Without OpenMP {#without-omp}
77 --------------
78 
79 If INCLUDE is not a system directory such as /usr/include/ or /usr/local/include/
80 then either it should be added to CPLUS_INCLUDE_PATH or at compilation using -I$INCLUDE. E.g.
81 
82  g++ -c -I$INCLUDE source.cpp
83 
84 When linking, GSL should be linked by added -lgsl.
85 If GSL's directory is not a system one or in LIBRARY_PATH then -L$GSLDIR should be added. E.g.
86 
87  g++ -L$GSLDIR source.o -lgsl
88 
89 With OpenMP {#with-omp}
90 -----------
91 
92 If OpenMP is to be used, then WITH_OMP should be set to 1,
93 -fopenmp -DWITH_OMP=$WITH_OMP used when compiling
94 and -lgomp when linking.
95 
96  g++ -c -fopenmp -DWITH_OMP=$WITH_OMP -I$INCLUDE source.cpp
97  g++ -L$GSLDIR source.o -lgsl -lgomp
